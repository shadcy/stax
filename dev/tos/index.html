4. T-OS UI Component Guidelines

Creating Windows

To register a new app inside the Window Manager (`wm.c`), use:

window_t *win = wm_add_window(100, 100, 400, 300, "My App", myapp_draw_window);
if (win) {
    win->key_event = myapp_key_event;
    win->update_client = myapp_update; // Optional for real-time loops
}

Drawing Primitives (RGB565)

Always draw relative to the `cx` and `cy` window offsets passed into the draw callback!

void myapp_draw_window(struct window *win, int cx, int cy, int cw, int ch) {
    // Fill window background
    fb_fillrect(cx, cy, cw, ch, COLOR_WHITE);
    
    // Draw text (using wm.h's global font engine)
    draw_text(cx + 10, cy + 10, "Hello World", COLOR_BLACK);
    
    // Draw custom RGB565 elements
    fb_fillrect(cx + 50, cy + 50, 20, 20, rgb565(255, 100, 100)); // Red square
}

Memory Safe Practices

Always check if `win->app_data` is initialized. If not, allocate it via `kmalloc()`. Since T-OS currently uses a Bump Allocator, do not repeatedly allocate large structures in update loops as `kfree()` may only reclaim memory effectively if it's the last block!
